Logic not wired in prototype
Real AI responses (Claude Sonnet 4.6)
Intake and check-in flows are scripted in the prototype. Real build uses Claude with mode-specific system prompts. Five system prompts total — one per mode. Tested independently.
Complex
Photo vision analysis
Claude vision reads body shape, skin, hair, outfit, grooming, overall vibe. Output feeds into score and audit. Rate-limited — vision calls are expensive. Cost model required before launch.
Complex
Score calculation engine
Claude returns category scores (style, grooming, vibe, social). Weighted average becomes overall score. Exponential difficulty curve — each point harder as you climb. Algorithm definition needed.
Complex
Profile persistence (Supabase)
profiles, scores, check_ins, photo_analyses tables. Profile JSON stored and passed with every check-in prompt. Score history tracked row by row.
Needs backend
Prism ID auth
Sign in / sign up shared across all Prism apps. No separate account per app. SSO-style within the suite. Supabase Auth or dedicated service TBD.
Needs backend
Rate limiting (Upstash Redis)
Free tier: 3 AI sessions/day. Pro: unlimited. Vision calls tracked separately — counted and cost-monitored. Limits shown in UI when approaching cap.
Needs backend
Mode-change re-score logic
Switching modes adjusts the difficulty weighting of historical scores retroactively or flags next check-in as a re-baseline. Behaviour needs definition.
